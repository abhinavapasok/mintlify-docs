The product guide explains how to use KomKits day to day — module by module, workflow by workflow. It is written for business users (owners, accountants, sales and purchase teams), not developers. If you are integrating with the API, see the API reference tab instead.

## How this guide is organized

Each module page follows the same structure so you always know where to look:

- **Overview** — what the module does and when to use it
- **Before you start** — permissions and settings you need
- **Key concepts** — terms and statuses used in the module
- **Common workflows** — step-by-step instructions for everyday tasks
- **Field reference** — what each important field means
- **FAQs** — answers to common questions
